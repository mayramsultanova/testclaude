# 🚀 Lavanda Sky - Серверное решение для автоматизации отзывов WB

## 📋 Что это?

Автоматический сервер который:
- ✅ Проверяет новые отзывы на WB каждые 10 минут
- ✅ Сохраняет их в локальную базу данных
- ✅ Позволяет просматривать БЕЗ лимитов API
- ✅ Генерирует ответы в стиле Lavanda Sky
- ✅ Публикует ответы на WB
- ✅ Работает 24/7

## 🎯 Как это решает проблему лимитов?

**Проблема:**
- Вы открываете панель → 2 запроса к WB API
- Обновляете → ещё 2 запроса
- = Ошибка 429

**Решение:**
- Сервер делает 1 запрос каждые 10 минут (автоматически)
- Данные сохраняются в базу
- Вы открываете сайт → данные из базы (0 запросов к WB!)
- Публикуете ответ → 1 запрос к WB
- = Никаких лимитов!

## 💡 Как это работает?

### Фоновая проверка (каждые 10 минут):
1. Сервер делает запрос к WB API
2. Получает новые отзывы
3. Сохраняет в базу SQLite
4. Ждёт 10 минут
5. Повторяет

### Когда вы открываете сайт:
1. Данные берутся из базы (НЕ из WB API!)
2. 0 запросов = нет лимитов
3. Можете открывать сколько угодно раз

### Когда публикуете ответ:
1. Только ОДИН запрос к WB API
2. Отзыв помечается как "опубликован"
3. Больше не показывается

## 🔧 Устранение проблем

### Ошибка 429 всё равно появляется?
- Увеличьте `CHECK_INTERVAL` до 15-20 минут
- Проверьте что сервер один (не запущено несколько копий)

### База данных не сохраняется?
- Railway сбрасывает файлы при перезапуске
- Решение: используйте Railway Database (PostgreSQL)
- Или подключите внешнюю БД

### Сервер не отвечает?
- Проверьте логи в Railway Dashboard
- Убедитесь что переменная `WB_API_TOKEN` установлена

## 🎨 Интеграция с Claude API (опционально)

Для генерации ответов через Claude (как в панели):

1. Добавьте переменную окружения:
```
ANTHROPIC_API_KEY=ваш_ключ_claude
```

2. В `wb_server_app.py` раскомментируйте секцию Claude API
